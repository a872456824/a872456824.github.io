多进程（面试必问）

进程：

  • 说法 ：

    • 程序在内存中的镜像；
    • 正在运行的程序（即程序实例化）

  • 什么是程序

    • 编译好的可执行的二进制文件，放在磁盘中
    • 在linux 中，程序是一个有X权限的普通文件
    • 应用 ： 程序的集合（程序 < 应用）

  • 什么是线程

进程接口

fork()

打不开请刷新一下页面

创建一个子进程，返回一个PID；

  • 调用fork 的是父进程，产生的是子进程

    子进程就是父进程的完全克隆体，把父进程整一份代码都克隆过去。

    但拷贝一开始并没有发生，只是登记一下，当内存发生变化（写入），拷贝才会发生（写拷贝），这样做可以节省内存空间。

    子进程刚产生的时候，所有的资源以及使用时间都会被清空归零。

  • 父进程与子进程的内存空间``是两个完全独立的进程空间，不过子进程的初始状态是父进程调用fork 时的状态，内容一模一样。

  • 子进程拥有独立且唯一的PID。

  • 子进程不会继承父进程的内存锁、异步IO操作、记录锁、计时器。

  • 返回值：返回两个PID，父进程中返回子进程的PID，子进程返回 0；

为什么会返回两个？因为fork产生子进程的时候，子进程跟父进程的状态是一模一样的，子进程也有fork，所以会返回两个，一个给子进程，一个给父进程。

子进程可以通过getpid() 与 getppid()得到自己与父进程的PID，但父进程无法通过其他函数得到子进程的PID，所以父进程只能通过fork 得到子进程的pid，而子进程不需要记住自己跟父进程的PID，所以fork 返回的是0.

  • 如果 pid < 0，则是fork出错，最常见的原因是内存不够了。

  • pid = 0 ： 子进程； pid > 0： 父进程

    父进程不一定会先执行，但大概率会先执行，是否可以先执行，是由内核基于调度的算法决定的。

    • cpu 会给每个进程一定时间的运行时间，谁先运行完谁先输出
    • 父进程在 fork 子进程之后，剩余时间不多，而子进程的时间完全没消耗，只能排在后面等待系统调度，所以会比父进程晚输出。
    • 而如果有信号中断了父子进程的运行，等内核重新调度的时候，父子进程谁优先就说不准了。

wait()

打不开请刷新一下页面

等待进子程发生变化，返回一个int 型的状态标识，0成功， 1中断

  • wstatus ： 存放子进程退出状态

  • 子进程状态： 正常死亡，异常死亡，唤醒。

  • 子进程死亡，父进程收尸，其实就是内核释放子进程的资源

  • 如果子进程成为“僵尸”，就会占用系统资源，可以在top中看到

  • 子进程状态一改变，wait 会立马得到返回值。

    子进程死了，父进程也死了，则不会产生“僵尸”

exec

打不开请刷新一下页面

用一个新的进程镜像替换当前的进程镜像。

fork复制产生的子进程，其后续的代码怎么运行，其实已经被父进程所规定，会跟父进程的后续代码一样，为了不跟父进程后续代码一样，有了exec一族函数

  • arg 是前面 path 的参数
  • execl()/execlp()/execle()：l是 list 列表的意思，包含着参数arg参数列表
  • execv()/execvp()/execvpe()：v是包含argv的意思，提供了一个数组，类似main函数的argv传递方式。
  • execle()/execvpe()：允许给可执行程序传递一个环境变量的参数envp
  • execlp()/execvp()/execvpe()：复制shell的过程，寻找可执行程序
