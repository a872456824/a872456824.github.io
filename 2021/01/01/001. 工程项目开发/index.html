C语言测试框架

发表于 2021-01-01 更新于 2021-03-09 分类于 工程项目 阅读次数： 阅读次数： Valine：

函数的声明与定义

  • 声明：告诉系统有这个东西
  • 定义：实现方式
  • 函数未声明和未定义暴露在两个时期
    • 函数未声明错误——编译过程（语法检查）
      • g++ -c \.cpp*生成编译后的对象文件
    • 函数未定义错误——链接过程
      • g++ \.o*链接生成可执行程序
  • 函数声明可以多次，但定义只能一次！

头文件与源文件

  • 规范
    • 头文件放声明，源文件放定义
    • 头文件与对应的源文件名称一致
  • 头文件中条件式编译，可以避免一次编译过程中重复包含头文件的问题

工程开发规范与静态链接库

  • #include后的双引号""改成尖角号<>

    • 双引号””：从执行代码所在目录下搜索
    • 尖角号<>：从系统库路径下搜索
    • 使用g++/gcc -I添加头文件路径到系统库路径下

  • 向上开发时

    • 给别人头文件(include文件夹)、源文件对应的对象文件的包(lib文件夹)即可

    • 对象文件打包

      • 静态链接库（.a）

      • ```c++
        // 打包
        ar -r libxxx.a header1.o header2.o header3.o
        // 链接 g++ *.o -L -l
        g++ test.o -L./lib -lxxx
        // xxx对应

  • 替换顺序

    • 从外层到里层，但遇到#或者##就会停止展开

    • 第一种：→#EarthQuake→”EarthQuake”

    • 第二种

      • 先替换第一层：→_ToStr(EarthQuake)→_ToStr(9.0)
      • 再替换第二层：_ToStr(9.0)→”9.0”

    • 嵌套定义：#define __ToStr(x) _ToStr(x)

    • 嵌套调用：__ToStr(EarthQuake)
