文件及目录

借助系统提供的底层服务，去完成想做的东西！

文件操作

预备基础

  1. cp ：先从磁盘中找到文件，再拷贝出来，最后写到磁盘中的目标文件
  2. mv：先从磁盘中找到文件，再拷贝出来写到磁盘中的目标文件，最后删除原文件
  3. cat： 把文件读取出来，放到标准输出文件

这几个文件都是跟读写文件相关的命令！而这些读写之前，都要先打开文件，即 open 函数！

open

打不开请刷新一下页面
  • flags：标志，即打开pathname 文件，用flag 标志的方式打开。
  • 放回一个新的文件描述符 或者 -1；
  • 返回的文件描述符可以在后续系统中被调用

被成功调用返回的文件描述符一定是当前进程中打开的最小数字！比如当前打开了 0、1、2、3、4 这五个文件，如果此时把 2 号文件关闭，然后打开第六个文件，则第六个文件返回的是 2 这个文件描述符，因为此时 2 是空置的描述符中最小的值。（也就是会补空！）

当打开了一个文件，创建了一个新的文件描述符的时候，系统内核中的全局表（管理所有文件的进程码）就会新增一条信息，表示该进程打开了该文件以及该文件的描述符、状态、偏移量等。

打开文件后，文件指针默认在文件头部

文件描述 [file description]

  • 每次调用open会创建一个新的open文件描述，它是系统全局文件表中的一条信息
    • 记录文件偏移量和文件的状态
  • [PS] 文件描述符是一个open文件描述的引用，不因pathname的改变而受影响
  • flags 中必须含有 O_RDONLY, O_WRONLY, 或者 O_RDWR。（只读、只写、可读可写）
  • O_CREAT : 创建
    • 底层是一个int型数据，叫做位掩码
      • 32位，可表示32种状态，每一位表示一种状态
      • 状态之间可使用与、或、异或的方式转换
  • O_TRUNC : 截断
  • O_DIRECT : 直接IO

直接 io： 直接写到内存中

缓冲 io ： 先放到缓存中，再存入内存中。（直到遇到回车或者程序结束符才会停止输出）

在磁盘中写的时候，也是类似的原理；输入一个字符，会先被系统放在缓冲中，如果在一段时间之内没有新数据写入，内核就会吧数据真正存放到内存中。（如果断电的话，缓存中的数据就会丢失！）

  • O_NONBLOCK : 非阻塞IO，如果没有拿到数据，则直接返回一个 -1

阻塞 io ：会浪费资源，一定要等到数据输入才肯罢休

非阻塞 io ：需要比较频繁的擦看数据，避免错过拿到数据的时间

  • O_TMPFILE: 系统运行时，会把数据放在这个临时文件夹中，当下次系统重启时，才会被清空

read

打不开请刷新一下页面

从该文件描述符fd 中尝试读取最多的到count字节的数据到 buff 中。

尝试读取，即有可能读不到那么多数据。

  • fd : 前面文件返回的描述符
  • ssize_t ：以_t 结尾的类型就是用户自定义类型

返回值 ssize_t：读取的字节数 或 -1

  • 猜测：也是基本类型之一，可能是long long，可能是int
  • 通过ctags一步步找具体类型：ctrl + ] 、ctrl + o
    • int [32位系统下]；long int [64位系统下]
  • 读取操作会从文件指针，即offset 位置开始。（offset 会自增，根据读取的进度自行增加，方便读取）
  • count = 0 的时候会返回 0 或者无效。
  • count > SSIZE_MAX 的时候会 返回一个未定义的值。

write

打不开请刷新一下页面

从该文件描述符fd 中尝试写入最多的到count字节的数据到 buff 中。

  • 对于seekable file（可搜索文件），将在offset处进行写入，并且offect将增加实际写入的字节数。 如果使用O_APPEND(追加)打开（2）文件，则在写入之前首先将offset设置为文件的末尾。

close

打不开请刷新一下页面

关闭文件描述符fd（内核关闭该文件）。

不关闭的话，可能会存在危害！！有些会自动关闭，有些不会，如果一堆文件不关闭，可能会使得系统崩溃

fopen

打不开请刷新一下页面

打开一个pathname文件，并关联到一个stream流（关联到一个字节河流）

这个fopen 才是标准io函数，之前的open 并不是一个标准io函数

  • FILE : 特殊文件指针，文件流
  • 参数 ： r、r+、w、w+、a、a+ 均是对一个文件流进行操作
  • 返回值 ：成功则返回一个FILE pointer文件指针， 失败则返回空指针，并将全局的错误变量建立起来

fread\fwrite：二进制流的IO

打不开请刷新一下页面
  • read 返回的是 ssize_t ，这里返回size_t
  • 从stream 中读写数据到 ptr 中，每次读取的大小为size，总共读取的次数为nmemb
  • 返回成功写到 item 的个数（字节数），如果发生错误，返回值则是一个小于size 的数字或者 0

fclose

打不开请刷新一下页面

关闭一个流stream。

fclose会刷新对应的stream（任何被写入的输出），因为标准IO里面的所有操作都是缓冲IO，所有需要刷新fflush，然后关闭该文件。

缓冲中时，没有任何权利去执行任何操作，只能等待。

所有到标准错误的输出stderr都会无缓冲的输出，不用任何等待。

中途忘记参数的时候，直接在函数位置，按 shift + k就可以跳到man手册
