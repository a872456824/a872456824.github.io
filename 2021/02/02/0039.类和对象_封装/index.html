构造函数与析构函数

由于没有学左值右值，所以目前这部分不是完整的知识。

  • 无返回值，且函数名与类名一样

  • 变量需要初始化，那对象也需要初始化，而初始化则是构造函数，析构函数则是销毁对象

  • 任何对象的生命周期：构造 ——使用 ——析构

  • 默认构造函数：编译器自动生成，没有参数的函数

    如果自己写了一个有参构造函数，那编译器的默认构造函数则已经被删掉不存在了。

    这个时候，如果还要使用默认构造函数，只能自己显示的写一个默认构造函数，即存在两个构造函数。

    仅仅是默认构造不见了，如果需要拷贝构造，仍然会自动生成。

  • 有参构造函数：

    • 转换构造函数：只有一个参数，相当于把一种类型转换成当前类型

    • 拷贝构造：传参类型与当前对象类型一致，把其他对象拷贝给当前对象

    • 移动构造：涉及右值

      拷贝构造与赋值运算符完全不等价。

  • 析构函数：回收资源，释放空间

在工程项目中，构造函数与析构函数一般会设计得非常非常的简单，会尽量避免在构造函数进行对大资源的申请，因为如果构造函数出 BUG 了，那 C++ 的异常捕获机制是很难捕获到的。

因此，一般会额外做一个伪构造函数跟伪析构函数（成员函数）去代替构造跟析构，或者说运用设计模式中的工厂设计模式。

三/五法则：

  1. 需要析构函数的类也需要拷贝构造函数和拷贝赋值函数
    • 一般需要析构函数的，都是需要动态申请资源的，而动态申请资源的拷贝一般都是深拷贝
  2. 需要拷贝操作的类也需要赋值操作，反之亦然
  3. 析构函数不能删除
  4. 如果有一个类有删除好的或不可访问的析构函数，那么其默认和拷贝构造函数会被定义为删除
  5. 如果一个类有const或引用成员，则不能使用合成的拷贝赋值操作

返回值优化（RVO）

打不开请刷新一下页面
  • 编译运行后发现 a 的地址与 temp 的地址一模一样，而且只调用了有参构造，没调用拷贝构造
    • 因为func（）中，temp 只是一个局部变量，其最终的结果均会传递给 a 对象，因此编译器就做了个优化，直接把 temp 这个局部变量中的 this 指针指向 a，这样所有的修改都变成是针对 a。即在func函数中，temp 变量的构造中，传给this 的是 a 的地址，而不是局部变量 temp 的地址
    • 这种优化，就是返回值优化。
  • 如果不用初始化列表显性的拷贝的时候，比如D类的d，则会调用其默认构造（D类的默认构造）。
  • 如果把A类的拷贝构造给注释掉的话，会自动生成一个默认拷贝构造，此时D类则是调用拷贝构造

对象初始化：

  • 默认初始化

  • 拷贝构造

  • 函数形参的初始化

    • 值传递，a 把自己的值拷贝给 b

  • 返回值优化（RVO）

    • 一般情况下，函数的返回值会对应一个临时的匿名变量，所以 return 的时候，相当于把 return 的值赋值给临时匿名变量

  • 中间的匿名变量没有任何价值，所以第一版优化就是直接把 temp_a 拷贝给 a

  • 因为 temp_a 是个局部变量，最终的所有操作的结果都会成为 a，所以第二版优化就是直接把temp_a看成是 a 的别名，直接对 a 进行操作，这样就只会调用一次构造函数，不会调用拷贝构造

在工程项目中，当实现完代码之后，因为并不知道到底实现了多少遍拷贝构造，所以不能改变拷贝构造的语义，因为改变了之后，编译器可能不知道改变了。即传入的值，一律不改变！

如果一个作用域内几个函数名字相同但是参数列表不同，称为函数重载。

  • 与返回值没有关系，编译器无法通过返回值区分函数版本
  • 冲突1：
    • 第二个函数有默认值，所以，第二个函数的参数，可以是一个，也可以是两个。如果主函数调用只有一个参数的函数，编译器不知道要调用第一个还是第二个。可以把第一个函数合并到第二个。

重载的意义：

  • 通过函数名对函数功能进行提示
  • 通过函数参数列表对函数的用法进行提示
  • 扩展已有的功能

特殊运算符重载：只支持类内重载

  1. [] ：数组对象

    • 返回值可以任意类型
    • 重载了[] 运算符之后，导致这个类下的所有对象都支持 [] 运算符
    • 外在表现得像一个数组，实际上还是一个对象！

  2. () ：函数对象

    • 外在表现得像一个函数，实际上还是一个对象！又名仿函数

  3. ->：指针对象、间接引用

    • 外在表现得像一个指针，实际上还是一个对象！

    • 返回的是一个地址，调用对象->相当于调用了其他对象，这样就可以使用该对象的成员方法

sort函数的使用：可调用对象

正常情况下，sort 的比较规则是传一个比较函数 cmp1

而其实也可以传递一个仿函数，函数名就是类的对象名字，如果直接类名()(已经重载了括号)，则是产生一个匿名对象，也可以传一个真正的对象。

仿函数的功能比正常的比较函数更加强大！！！

  • 比如可以增加一个整型变量z，用来判断是从大到小排序，还是从小到大排序，图中默认从小到大。这种操作用普通的比较函数是做不到的。
